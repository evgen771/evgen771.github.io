Для работы `Beamer` через `Pandoc` обычно не нужно устанавливать пакеты по отдельности, так как в современных дистрибутивах (как ваш `TeX Live 2026`) они идут в комплекте. 

Однако важно знать, какие именно компоненты задействуются, чтобы устранить ошибки, если они возникнут.

## Ядро Beamer

Сам класс `beamer` зависит от нескольких ключевых пакетов, которые должны быть в системе: 

* `pgf`: графический движок для отрисовки элементов интерфейса и анимаций.
* `xcolor`: для управления цветами тем и текста.
* `hyperref`: обеспечивает интерактивность, навигацию и ссылки в `PDF`.
* `amsfonts`, `amssymb`, `amsmath`: для корректного отображения математических формул на слайдах. 

## Пакеты, которые Pandoc добавляет сам

При конвертации `Pandoc` использует стандартный шаблон, который автоматически подключает следующие пакеты (если они есть в системе): 

* `unicode-math`: для поддержки современных шрифтов (при использовании `XeLaTeX/LuaLaTeX`).
* `graphicx`: если в презентации есть изображения.
* `listings` или `fancyvrb`: если вы используете блоки кода с подсветкой синтаксиса.
* `booktabs`, `longtable`: если в презентации есть таблицы.
* `babel` или `polyglossia`: для корректных переносов и перевода стандартных слов (например, «Содержание» вместо «Contents»). 

## 3. Что делать, если Pandoc выдает ошибку "package not found"?

Если при запуске команды вы видите ошибку о недостающем `.sty` файле, в `TeX Live` его можно установить одной командой:

`tlmgr install beamer pgf xcolor ms`

(Где `ms` содержит необходимые дополнения вроде `ragged2e` для выравнивания текста).

Совет по локализации: 

Для русского языка в `Beamer` через `Pandoc` критически важны пакеты `translator` (входит в `beamer`) и правильная настройка `babel` через переменную `-V lang=ru-RU`.
