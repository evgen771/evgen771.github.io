Чтобы добавить путь в переменную `PATH` в FreeBSD с использованием файла `~/.shrc`, нужно понимать, что этот файл используется оболочками, совместимыми с `sh` или `bash`, тогда как для `csh` или `tcsh` используется `~/.cshrc`.

Если вы используете **оболочку `sh` или `bash`**, то:

1. Откройте или создайте файл `~/.shrc` в домашней директории:

   ```sh
   nano ~/.shrc
   ```

2. Добавьте строку для расширения `PATH`. 

   Например, чтобы добавить директорию `~/bin`:

   ```sh
   export PATH="$PATH:$HOME/bin"
   ```

3. Сохраните файл и перезагрузите конфигурацию:

   ```sh
   source ~/.shrc
   ```

   Если же вы используете **`csh` или `tcsh`**, то нужно редактировать `~/.cshrc`:

   ```csh
   set path = ($path $HOME/bin)
   ```

Также убедитесь, что переменная `PATH` не переопределяется в других файлах инициализации, таких как `~/.profile` (для `sh`) или `~/.login` (для `csh`).

**Пример для `~/.shrc`:**

```sh
# Добавляем ~/bin и ~/.local/bin в PATH
export PATH="$PATH:$HOME/bin:$HOME/.local/bin"

# Добавляем ~/bin в PATH
export PATH="/home/jenit/bin:$PATH"

# Устанавливаем редактор по умолчанию
#export EDITOR=nano
#export VISUAL=nano
export EDITOR=/usr/bin/ee
export VISUAL=/usr/bin/ee

# Цветной вывод для ls
alias ls='ls -G'
```

После этого все скрипты из `~/bin` будут доступны как команды из любого места в терминале.

> Обратите внимание: если вы используете `zsh`, файл будет `~/.zshrc`, а не `~/.shrc`. Убедитесь, что используете правильный конфигурационный файл для вашей оболочки.

Если вы не уверены, какая оболочка у вас активна, выполните:

```sh
echo $SHELL
```
