#lualatex #xelatex #luatex #xetex

Emacs считается одной из лучших сред для работы с $\LaTeX$ благодаря мощным расширениям, которые превращают текстовый редактор в полноценную IDE для верстки. 

## Основные инструменты

Для комфортной работы в Emacs обычно используют связку следующих пакетов:

* AUCTeX: Самый популярный и мощный пакет для $\LaTeX$ в Emacs. Он обеспечивает:
* Умное автодополнение команд и окружений.
   * Быструю компиляцию документа (обычно C-c C-c) и просмотр PDF.
   * Сворачивание кода (folding) и проверку синтаксиса.
* Preview-latex: Входит в состав AUCTeX. Позволяет отображать формулы и изображения прямо в окне редактора поверх кода.
* RefTeX: Вспомогательный режим для управления ссылками, цитатами и оглавлением.
* CDLaTeX: Легковесный пакет для сверхбыстрого ввода математических формул с помощью «горячих» клавиш и шаблонов. 

## Полезные сочетания клавиш (в режиме AUCTeX)

* C-c C-e — вставить окружение (\begin{...} \end{...}).
* C-c C-s — вставить секцию (главу, раздел).
* C-c C-c — запустить компиляцию или открыть просмотрщик PDF.
* C-c C-m (или C-c RET) — вставить макрос/команду. 

## Альтернативный подход: Org-mode
Многие пользователи предпочитают писать текст в Org-mode и экспортировать его в $\LaTeX$. Это позволяет использовать более простой синтаксис для заголовков и списков, сохраняя возможность вставки «чистого» $\LaTeX$-кода и формул там, где это необходимо. 

## Особенности работы с русским языком

Для корректной поддержки кириллицы в документах (независимо от редактора) в преамбуле стоит использовать современные движки и пакеты:  

* Для классического pdfLaTeX: \usepackage[utf8]{inputenc} и \usepackage[T2A]{fontenc}.
* Для XeLaTeX/LuaLaTeX: рекомендуется использовать пакет fontspec для прямой работы со шрифтами системы. 

---

Для XeLaTeX и LuaLaTeX использование fontspec — это стандарт индустрии. Эти движки работают напрямую со шрифтами формата OpenType (OTF) и TrueType (TTF), установленными в вашей системе, избавляя от необходимости мучаться с конвертацией шрифтов под TeX.
Вот как выглядит минимальный рабочий шаблон для Emacs (сохраните файл с расширением .tex):

\documentclass{article}

% Основной пакет для управления шрифтами
\usepackage{fontspec}

% Настройка шрифтов для русского языка
\setmainfont{Times New Roman} % Или другой шрифт, который есть в системе
\setsansfont{Arial}
\setmonofont{Courier New}

% Пакет для правильных переносов в русском языке
\usepackage[russian]{polyglossia}
\setmainlanguage{russian}
\setotherlanguage{english}

\begin{document}
Привет, Emacs! Теперь я использую системные шрифты через XeLaTeX.
\end{document}

## Как заставить Emacs использовать XeLaTeX по умолчанию

AUCTeX по умолчанию часто запускает обычный pdflatex. Чтобы переключиться на xelatex, есть два способа:

   1. Локально для файла (в конце документа):
   Добавьте эти строки в самый конец вашего .tex файла:
   
   %%% Local Variables:
   %%% mode: latex
   %%% TeX-engine: xetex
   %%% End:
   
   После этого переоткройте файл (M-x revert-buffer), и Emacs сам поймет, каким движком его собирать.
   2. Глобально в конфиге (init.el):
   
   (setq-default TeX-engine 'xetex)
   
   
## Почему LuaLaTeX иногда лучше?

Если вы планируете писать сложные макросы или работать с очень объемными документами, LuaLaTeX считается более современным и стабильным (хотя и чуть более медленным). Настройка в fontspec для него идентична, нужно только сменить TeX-engine на luatex.

Для интеграции просмотра PDF прямо в интерфейс Emacs лучше всего подходит пакет PDF Tools. В отличие от стандартного doc-view, он отрисовывает страницы на лету, позволяет выделять текст, искать по документу и поддерживает SyncTeX (быстрый переход от кода к PDF и обратно). 

## 1. Установка зависимостей (в системе)

Перед установкой пакета в Emacs необходимо установить системную библиотеку poppler, на которой он базируется.  

* Ubuntu/Debian: sudo apt install libpoppler-glib-dev libpoppler-private-dev zlib1g-dev
* macOS: brew install poppler automake pkg-config
* Windows: рекомендуется использовать MSYS2 и установить mingw-w64-x86_64-poppler. 

## 2. Установка и настройка в Emacs

После установки системных зависимостей выполните следующие шаги в Emacs:

   1. Установите пакет: M-x package-install RET pdf-tools RET.
   2. Скомпилируйте серверную часть: M-x pdf-tools-install RET. 
   Emacs предложит собрать программу epdfinfo. 
   Согласитесь, дождитесь завершения и перезапустите Emacs.
   3. Добавьте базовую настройку в init.el: 

```latex
(use-package pdf-tools
  :ensure t
  :config
  (pdf-tools-install)                   ; Инициализация при запуске
  (setq-default pdf-view-display-size 'fit-page) ; PDF сразу по размеру окна
  (setq TeX-view-program-selection '((output-pdf "PDF Tools")) ; Использовать для LaTeX
        TeX-view-program-list '(("PDF Tools" TeX-pdf-tools-sync-view))
        TeX-source-correlate-start-server t) ; Для обратного поиска (SyncTeX)
  (add-hook 'TeX-after-compilation-finished-functions
            #'TeX-revert-document-buffer)) ; Автообновление PDF после компиляции
```

## 3. Основные возможности связки

* Прямой поиск: C-c C-v в коде $\LaTeX$ откроет PDF на соответствующей странице.
* Обратный поиск: C-mouse-1 (Ctrl + левый клик) в окне PDF переместит курсор на нужную строку в коде.
* Автообновление: как только вы пересоберете документ (C-c C-c), окно PDF внутри Emacs обновится автоматически без потери позиции. 

Хотите узнать, как включить автоматическую компиляцию «на лету», чтобы PDF обновлялся сразу после сохранения файла?

---

Для реализации автоматической компиляции при сохранении файла (Auto-save/Auto-compile) в Emacs чаще всего используют пакет latex-preview-pane или связку AUCTeX с хуками.
Самый надежный способ — настроить хук after-save-hook, который будет запускать компиляцию каждый раз, когда вы нажимаете C-x C-s.

## 1. Настройка в init.el

Добавьте этот код в ваш файл конфигурации:

```
(defun my/latex-compile-on-save ()
  "Автоматически запускает компиляцию при сохранении .tex файла."
  (when (eq major-mode 'latex-mode)
    (TeX-command-sequence '("LaTeX") t)))

(add-hook 'after-save-hook #'my/latex-compile-on-save)
```
## 2. Продвинутый вариант через пакет latex-preview-pane

Если вы хотите видеть превью «как в Overleaf» (справа от текста), можно установить этот пакет:

   1. Установите: M-x package-install RET latex-preview-pane RET.
   2. Активируйте в файле: M-x latex-preview-pane-mode.
   3. Теперь при каждом сохранении окно превью будет обновляться автоматически.

## 3. Важные советы для работы «на лету»

* Игнорирование ошибок: Чтобы Emacs не прерывал процесс компиляции на каждой опечатке, в настройках AUCTeX обычно включен режим TeX-interactive-mode (выключите его, чтобы компиляция шла в фоновом режиме).
* Magic Comments: Чтобы автоматика знала, что использовать именно XeLaTeX, убедитесь, что в конце файла стоят локальные переменные (которые мы обсуждали ранее):

%%% Local Variables:
%%% mode: latex
%%% TeX-engine: xetex
%%% End:
