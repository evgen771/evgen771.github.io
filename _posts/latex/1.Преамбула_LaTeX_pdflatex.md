### Преамбула LaTeX: что это и зачем нужна


**Преамбула** — это начальная часть LaTeX‑документа, расположенная *между* командой объявления класса документа (`\documentclass`) и началом основного текста (`\begin{document}`).

Именно здесь задаются фундаментальные параметры оформления, подключаются пакеты, определяются пользовательские команды и настраивается поведение документа.

### Структура преамбулы

Типичная преамбула выглядит так:

```latex
\documentclass[опции]{класс}
% Здесь идут настройки, пакеты, команды
\begin{document}
% Основной текст документа
\end{document}
```

### Основные элементы преамбулы

1. **Объявление класса документа** 

   ```latex
   \documentclass[12pt,a4paper]{article}
   ```
   - `article`, `book`, `report`, `letter` — основные классы;
   - опции в квадратных скобках задают размер шрифта, формат бумаги и др.

2. **Подключение пакетов** 

   ```latex
   \usepackage[utf8]{inputenc}     % кодировка текста
   \usepackage[russian]{babel}    % поддержка русского языка
   \usepackage{amsmath}          % расширенные математические формулы
   \usepackage{graphicx}         % вставка изображений
   ```
   Пакеты расширяют функционал LaTeX: от поддержки языков до сложных графиков.

3. **Настройка параметров страницы** 

   ```latex
   \usepackage[left=2cm,right=2cm,top=2cm,bottom=2cm]{geometry}
   \pagestyle{plain}              % стиль колонтитулов
   ```

4. **Определение пользовательских команд** 

   ```latex
   \newcommand{\R}{\mathbb{R}}     % множество действительных чисел
   \newcommand{\limn}{\lim_{n\to\infty}} % часто используемый предел
   ```
   Упрощает ввод повторяющихся конструкций.

5. **Метаданные документа** 

   ```latex
   \title{Название работы}
   \author{Иванов И.И.}
   \date{\today}
   ```
   Будут использованы в титульном листе (если он создаётся).

6. **Глобальные настройки** 

   ```latex
   \linespread{1.3}             % межстрочный интервал
   \setlength{\parindent}{1.25cm} % абзацный отступ
   ```

### Важные правила работы с преамбулой

- **Порядок команд имеет значение!** Некоторые пакеты (например, `hyperref`) нужно подключать в конце преамбулы.
- **Комментарии** (строки после `%`) помогают структурировать код.
- **Избегайте избыточности:** подключайте только нужные пакеты — это ускоряет компиляцию.
- **Для кириллицы** обязательно укажите:

  ```latex
  \usepackage[T2A]{fontenc}
  \usepackage[utf8]{inputenc}
  \usepackage[russian]{babel}
  ```

### Пример полной преамбулы

```latex
\documentclass[12pt,a4paper]{article}
\usepackage[T2A]{fontenc}
\usepackage[utf8]{inputenc}
\usepackage[russian]{babel}
\usepackage{amsmath,amssymb}
\usepackage{graphicx}
\usepackage[left=2cm,right=2cm,top=2cm,bottom=2cm]{geometry}

\title{Исследование функций}
\author{Петров А.С.}
\date{\today}

\newcommand{\N}{\mathbb{N}}
\newcommand{\Z}{\mathbb{Z}}
```

### Почему преамбула важна?

- **Централизация настроек:** все глобальные параметры собраны в одном месте.
- **Повторное использование:** готовую преамбулу можно копировать в новые документы.
- **Читаемость кода:** разделение настроек и содержимого упрощает редактирование.
- **Предотвращение ошибок:** правильная последовательность пакетов исключает конфликты.
